# CLAUDE.md — components/
_Last updated: 2026-05-21. If patterns in the codebase contradict this file, report it._

## § OWNS
React components, grouped by feature (`orders/`, `clients/`, `autoparts/`, `reservations/`, `purchases/`, `analytics/`, `admin/`, `general/`, `activity/`) plus `layout/` (sidebar, breadcrumbs) and `ui/` (shadcn primitives). Must not contain: route definitions, Prisma calls, server-only secrets, business calculations that aren't UI-local (extract to `lib/`).

## § CONVENTIONS
- Filename = PascalCase + `.tsx` (e.g. `OrdersTable.tsx`, `OrderPaymentModal.tsx`). One component per file. Default export.
- `components/ui/*` — generated by `npx shadcn@latest add <name>`. Config: [components.json](components.json) (style: new-york, base: neutral, css vars). **Do not hand-edit shadcn primitives** unless adding a project-specific variant — re-run shadcn or wrap.
- Icons: `lucide-react`. Class merging: `cn()` from `@/lib/utils`.
- Toasts: `import { toast } from 'sonner'`. Errors → `toast.error(...)`, success → `toast.success(...)`. The `<Toaster />` is already mounted in `app/layout.tsx`.
- All client components start with `'use client';` on line 1.
- Russian copy in JSX; English in identifiers.

## § CURRENT PATTERNS ✓
- Feature table component (canonical: [components/orders/OrdersTable.tsx](components/orders/OrdersTable.tsx)):
  - `'use client'`, local `useState` for `data`, `loading`, filter values, and `selected*` for modals.
  - Data fetch via `fetch('/api/<resource>?...')` inside `fetchData()`; called from `useEffect` keyed on filter deps.
  - Modal siblings (`OrderModal`, `OrderEditModal`, `OrderDetailsModal`, `OrderPaymentModal`) co-located in the same feature folder.
  - Mobile-aware: scroll listener hides filters on small screens (≥768 desktop).
- Shared layout primitives: `<Sidebar />` ([components/layout/sidebar.tsx](components/layout/sidebar.tsx)) defines the `navItems` array — to add an admin section, add an entry there + create `app/dashboard/<slug>/page.tsx`.
- Admin gating: wrap protected client trees in `<PinProvider>` (already in `app/dashboard/layout.tsx`); use `<PinCodeModal />` for elevated actions.
- `@/components/global-error-handler` + `@/components/ui/sonner` are mounted globally — don't re-mount.

## § LEGACY PATTERNS ✗ (DO NOT REPLICATE)
None unique to this directory. (Inline `Record<string, unknown>` types around `fetch` responses are common — fine for now; if a shape stabilises, move it into `app/types/<feature>.ts`.)

## § SUBAGENT CONTEXT
React 19 components for a Russian-language admin/storefront. Two kinds of files: shadcn primitives in `components/ui/*` (do not hand-edit — regenerate via `npx shadcn@latest add`) and feature components in `components/<feature>/*` (PascalCase, default export, one per file). Feature components are nearly always `'use client'` and fetch directly from `/api/*` route handlers with vanilla `fetch` + `useState` + `useEffect` — there is no React Query, SWR, or global state library. Errors → `toast.error()` from `sonner`. Class merging via `cn()` from `@/lib/utils`. Icons from `lucide-react`. To add a new admin section: append to the `navItems` array in `components/layout/sidebar.tsx` AND create `app/dashboard/<slug>/page.tsx`.
